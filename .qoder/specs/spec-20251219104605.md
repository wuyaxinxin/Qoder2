# 规则文件解析任务规范

## 任务概述
解析并说明 `.qoder/rules/dd.md` 文件的内容和作用。这是一个 Qoder 规则配置文件，用于在特定条件下自动应用代码生成规则。

## 文件内容分析

### 文件路径
`/Users/admin/Documents/testQoder/Qoder2/.qoder/rules/dd.md`

### 文件结构

```markdown
---
trigger: model_decision
description: 在生成java文件时生效
---
文件最上面加上"自由万岁"
```

### 组成部分详解

#### 1. Front Matter（前置元数据）
使用 YAML 格式，位于 `---` 分隔符之间：

- **trigger: model_decision**
  - 触发类型：模型决策型
  - 含义：由 AI 模型根据上下文自动判断是否应用此规则
  - 区别于 `always_on`（始终生效）和 `manual`（手动触发）

- **description: 在生成java文件时生效**
  - 描述信息：明确此规则的适用场景
  - 提示：当生成 Java 文件时，此规则应该被激活

#### 2. 规则内容
```
文件最上面加上"自由万岁"
```
- 具体指令：在生成的 Java 文件顶部添加注释 "自由万岁"
- 实际效果：Java 文件开头会包含 `// 自由万岁` 或 `/* 自由万岁 */`

## 规则类型对比

| 触发类型 | 触发条件 | 示例文件 |
|---------|---------|---------|
| `always_on` | 始终自动生效 | aa.md, ff.md, tool.md |
| `model_decision` | AI 模型根据场景判断 | dd.md (Java 文件时) |
| `manual` | 用户手动激活 | ee.md (八阵图) |

## 实际应用场景

当用户请求生成 Java 文件时，Qoder 会：
1. 检测到这是 Java 文件生成任务
2. 自动激活 dd.md 规则
3. 在文件顶部添加 "自由万岁" 注释
4. 同时应用 always_on 规则（如 "始终生效"）

## 规则优先级

根据当前系统配置，生成 Java 文件时会应用：
1. **始终生效规则**（aa.md）：顶部添加 "始终生效"
2. **模型决策规则**（dd.md）：顶部添加 "自由万岁"
3. **会话结束规则**（ff.md）：输出 "天下为公"

## 总结

`.qoder/rules/dd.md` 是一个**条件触发型规则配置文件**，专门用于 Java 文件生成场景，确保所有自动生成的 Java 文件顶部都包含特定的注释标记。
