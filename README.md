# Qoder2# 学生管理系统 (Student Management System)

这是一个功能完整的学生信息管理系统，使用Python开发，提供了学生信息管理、成绩管理、查询统计、数据持久化等核心功能。

## 🚀 主要功能

### 学生管理
- ✅ 添加学生信息（姓名、年龄、学号、专业）
- ✅ 删除学生记录
- ✅ 修改学生信息
- ✅ 查看学生详情
- ✅ 显示所有学生列表
- ✅ 按条件搜索学生

### 成绩管理  
- ✅ 添加学生成绩
- ✅ 修改成绩记录
- ✅ 删除成绩记录
- ✅ 查看学生成绩详情
- ✅ 成绩统计（平均分、最高分、最低分）

### 查询统计
- ✅ 成绩排名（按平均分排序）
- ✅ 优秀学生列表（平均分≥85且无不及格科目）
- ✅ 系统统计信息
- ✅ 及格/不及格科目统计

### 数据管理
- ✅ 数据持久化（JSON格式）
- ✅ 自动保存和加载
- ✅ 数据备份功能
- ✅ 数据导出（JSON/CSV格式）
- ✅ 数据完整性验证

## 📁 项目结构

```

├── student.py                    # 学生类定义
```

### 基本操作流程

1. **添加学生** → 输入学生基本信息
2. **添加成绩** → 为学生录入各科成绩
3. **查询统计** → 查看排名和统计信息
4. **数据管理** → 备份和导出数据

## 💻 代码示例

### 基本使用

```python
from student_management_system import StudentManagementSystem

# 创建管理系统实例
system = StudentManagementSystem()

# 添加学生
system.add_student("张三", 20, "2021001", "计算机科学")
system.add_student("李四", 19, "2021002", "数学")

# 添加成绩
system.add_student_score("2021001", "高等数学", 95)
system.add_student_score("2021001", "程序设计", 88)

# 查询学生
student = system.get_student("2021001")
print(student.get_student_info())

# 获取排名
top_students = system.get_top_students(5)
for i, (student, avg_score) in enumerate(top_students, 1):
    print(f"{i}. {student.name}: {avg_score:.2f}分")

# 保存数据
system.save_data()
```

```json
        "程序设计": 88.0,
        "英语": 76.0
      },
      "created_time": "2025-10-13T10:30:00"

  },
  "metadata": {
    "last_updated": "2025-10-13T10:35:00",
    "total_students": 1
  }
}
```

## 🔧 配置选项

### 数据文件配置
- 默认数据文件： `students_data.json`
- 备份目录： `data/backups/`
- 导出目录： `data/exports/`
- 日志目录： `data/logs/`

### 成绩设置
- 成绩范围：0-100分
- 及格分数：60分（可配置）
- 优秀学生标准：平均分≥85分且无不及格科目

### 学号格式
- 标准格式：7位数字（如：2021001）
- 前4位：入学年份（20XX）
- 后3位：学生序号（001-999）

## 🚨 注意事项

1. **数据安全**：
   - 建议定期备份数据
   - 重要操作前先备份
   - 避免直接编辑JSON数据文件

2. **学号管理**：
   - 学号必须唯一，系统会自动检查
   - 学号格式必须符合规范（2021001）
   - 不支持修改已存在的学号

3. **成绩录入**：
   - 成绩范围：0-100分
   - 支持小数分数（如：85.5）
   - 同一科目成绩会被覆盖更新

4. **系统限制**：
   - 内存中操作，不适合超大数据量
   - 建议学生数量控制在10000以内
   - 单个学生科目数量建议不超过50个

## 🤝 开发信息

- **开发语言**：Python 3.6+
- **架构模式**：面向对象设计
- **数据存储**：JSON文件
- **用户界面**：命令行界面（CLI）
- **测试框架**：unittest

## 📈 扩展功能建议

1. **Web界面**：可以添加Flask/Django web界面
2. **数据库支持**：支持MySQL、SQLite等数据库
3. **批量导入**：支持从Excel/CSV批量导入学生数据
4. **成绩分析**：添加更多统计图表和分析功能
5. **用户权限**：添加教师、管理员等不同角色权限
6. **API接口**：提供RESTful API接口
